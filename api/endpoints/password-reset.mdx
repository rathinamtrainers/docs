LearnFlow's password reset flow uses two endpoints. First, request a reset link by email. Then, submit your new password using the token from that email. Both calls are unauthenticated — they're designed for users who cannot log in.

---

## POST /auth/password-reset/request

Send a password reset email to the address associated with your account.

**Endpoint**

```
POST https://api.learnflow.io/auth/password-reset/request
```

**Authentication:** None required.

### Request body

<ParamField body="email" type="string" required>
  The email address associated with your account.
</ParamField>

### Request example

```http
POST /auth/password-reset/request HTTP/1.1
Host: api.learnflow.io
Content-Type: application/json

{
  "email": "learner@example.com"
}
```

### Response: 202 Accepted

Always returned, regardless of whether the email exists (privacy protection).

```json
{
  "message": "If an account exists for this email, a reset link has been sent."
}
```

<Note>
  The API returns `202` even if the email is not registered. This prevents account enumeration.
</Note>

---

## POST /auth/password-reset/confirm

Set a new password using the token from your reset email.

**Endpoint**

```
POST https://api.learnflow.io/auth/password-reset/confirm
```

**Authentication:** None required.

### Request body

<ParamField body="token" type="string" required>
  The reset token from the email link.
</ParamField>

<ParamField body="new_password" type="string" required>
  Your new password. Must meet the same complexity requirements as registration: at least 8 characters, one uppercase letter, one digit, and one special character.
</ParamField>

### Request example

```http
POST /auth/password-reset/confirm HTTP/1.1
Host: api.learnflow.io
Content-Type: application/json

{
  "token": "abc123resettoken",
  "new_password": "NewSecurePass2!"
}
```

### Response: 200 OK

```json
{
  "message": "Password updated. Please log in."
}
```

### Error responses

| Status | Message | Cause |
|--------|---------|-------|
| `400` | `"This password reset link has expired."` | Token is older than 1 hour |
| `400` | `"This reset link has already been used."` | Token was already consumed |
| `400` | `"This link is no longer valid. A newer reset link has been sent."` | A newer reset was requested, superseding this token |
| `422` | `"Your new password must be different from your current password."` | The submitted password matches your existing password |
| `422` | Password complexity errors | New password does not meet the required complexity rules |

<Note>
  Successfully confirming a password reset invalidates **all** active sessions for your account across all devices.
</Note>
